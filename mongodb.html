Core Concepts

MongoDB is a NoSQL, document-oriented database. Instead of tables and rows, it uses collections and documents.

  • Document: A BSON (binary JSON) object. This is the basic unit of data. It's a set of key-value pairs, much like a JSON object. 
    { "_id": ObjectId("..."), "name": "Alice", "age": 30, "city": "New York" }
  • Collection: A group of documents. This is the equivalent of a table in a relational database, but it does not enforce a strict schema.
  • Flexible Schema: Different documents in the same collection can have different fields. This is powerful for evolving applications.

Basic CRUD Operations

CRUD (Create, Read, Update, Delete) operations are the bread and butter of any database.

// Use a specific database
use("myDatabase");

// CREATE: Insert a single document
db.users.insertOne({ "name": "Bob", "age": 25, "status": "active" });

// READ: Find documents
db.users.find({ "status": "active" }); // Find all users with status "active"
db.users.findOne({ "name": "Bob" });   // Find the first user named "Bob"

// UPDATE: Modify a document
db.users.updateOne(
    { "name": "Bob" },                  // Filter: Find documents to update
    { $set: { "age": 26 } }             // Update: Set the age to 26
);

// DELETE: Remove documents
db.users.deleteOne({ "name": "Bob" });

Key Concept: Indexing

Indexing is the most important concept for MongoDB performance. Without indexes, MongoDB must perform a collection scan (reading every document) to find matching query results.

An index stores the value of a specific field (or fields) in a pre-sorted order, allowing MongoDB to find data extremely quickly.

Single-Field Index

Used to speed up queries on a single field.

// Creates an ascending (1) index on the 'name' field
db.users.createIndex({ "name": 1 });

// This query will now use the index and be very fast
db.users.find({ "name": "Alice" });

Compound Index

An index on multiple fields. The order of fields matters.

// Creates an index on 'status' (ascending) then 'age' (descending)
db.users.createIndex({ "status": 1, "age": -1 });

// This query can be fully supported by the index
db.users.find({ "status": "active" }).sort({ "age": -1 });

// This query can also use part of the index ("status")
db.users.find({ "status": "pending" });

Key Concept: Aggregation Framework

The Aggregation Framework is MongoDB's way of performing data processing and analysis, similar to GROUP BY in SQL or a Pig/Hive script.

It uses a pipeline concept, where documents pass through a series of stages, each performing a transformation.

Example Pipeline

Let's say we have a products collection. We want to find the average price of all "Electronics" products, grouped by their sub-category.

db.products.aggregate([
    {
        // Stage 1: Filter the documents (like WHERE)
        $match: { "category": "Electronics" }
    },
    {
        // Stage 2: Group by sub_category and calculate average (like GROUP BY)
        $group: {
            "_id": "$sub_category",  // Group by this field
            "averagePrice": { $avg: "$price" },
            "itemCount": { $sum: 1 }
        }
    },
    {
        // Stage 3: Sort the results by average price
        $sort: { "averagePrice": -1 } // -1 for descending
    }
]);
